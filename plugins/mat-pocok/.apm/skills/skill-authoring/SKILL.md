# Skill Authoring

Use this skill to draft a new `SKILL.md` or tighten an existing one for the `mat-pocok` package.

## Workflow

1. State the behavior the skill should help with in one sentence.
2. Limit the skill to one user goal or workflow.
3. Put setup, steps, and guardrails in the body.
4. Include only commands that are useful in practice.
5. Keep examples small enough to copy into a terminal or editor.

## Guardrails

- Avoid combining unrelated workflows in the same skill.
- Prefer direct instructions over long background explanations.
- Make the skill usable without reading sibling files.
