Support layer

Trust, install, validate, and publish the marketplace.

How the pieces connect
1. Root marketplace manifest apm.yml defines the marketplace owner, homepage, plugin root, build tag pattern, and the package list.
2. Generated Claude artifact .claude-plugin/marketplace.json mirrors the manifest so Claude tooling can consume the marketplace.
3. Plugin package plugins/factory/apm.yml exposes the package metadata and targets claude + codex.
4. Runtime dependency mcp-remote bridges agents to Linear’s official MCP endpoint using stdio.
Color cues
Pink = root manifest, purple = Claude artifact, teal = plugins, gold = package.
